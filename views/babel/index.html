# babel 是什么？

babel 是一个 JavaScript 编译器，因为部分 JavaScript 语法和 API 并没有得到所有环境的支持 ，所以需要使用 babel 将未支持的语法和 API 编译成支持的语法和 API。
babel 能做的事情：

  • 语法转换
  • 通过 Polyfill 方式在目标环境中添加缺失的特性 （通过引入第三方 polyfill 模块，例如 core-js (opens new window)）
  • 源码转换

# babel 的工作流程

babel 的工作流程基本分为三步

  • 解析（parse）
  • 转换（transform）
  • 生成（generate）

解析 -> 转换 > 生成

# 语法转换

语法转换是指将 JavaScript 高版本的语法转换成低版本的语法，从而达到每个环境都能完美运行。
在进行转换之前需要先安装@babel/cli和 @babel/core这两个 npm 包。

# @babel/cli & @babel/core

@babel/cli是一个命令行工具，我们需要用它来执行命令，编译文件。
@babel/core是 babel 的核心代码包，它提供了解析和转换的 api 供插件使用 。

npm install --save-dev @babel/core @babel/cli
1

安装完成后就可以使用 babel 的命令编译文件了。
babel 命令行常用参数

  • --out-file：指定编译结果的输出文件
  • --out-dir：指定编译结果的输出目录
  • --plugins：插件
  • --presets：预设
  • --help：帮助
  • ...
{
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
    "build": "babel ./src --out-file ./compiled.js"
  }
}

# presets（预设）

有没有发现每一种语法都需要单独的插件进行转换，如果每个都需要手动添加插件，这样的需要消耗的时间和精力是较大的，所以 babel 除了有 plugins配置之外还有 presets，presets可以理解为插件的集合包，将众多插件集中在一个包里，应付一些常见的场景。
安装 @babel/preset-env 预设

npm install --save-dev @babel/preset-env

# polyfill（垫片）

polyfill是一些较新的 JavaScript API 的旧版本代码实现，为了抹平不同环境的 API 支持度差异。

# babel/polyfill

babel/polyfill 是一个有大部分新API``polyfill的库
安装，注意这里要添加到 dependencies 中去，因为转换过后也需要依赖 polyfill

npm install --save @babel/polyfill
1

在所有代码之前引入 polyfill

import '@babel/polyfill'

let str = 'str'
const fun = function fun() {}
const promise = new Promise()

@babel/polyfill 有两个问题

  • 文件体积
  • 修改全局方法原型

我们实际上这里只需要转换 **Promise **，但是我们这里导入了整个 **polyfill **，这样势必会增加文件体积
@babel/polyfill 这个包是在全局的对象上修改了方法，可能导致出现冲突和其他问题。

# core-js 搭配 babel/preset-env

core-js 是一个 polyfill 库，大部分 API 的 polyfill 都在其中
为了解决 @babel/polyfill 全量引入的问题，我们可以使用@babel/preset-env配合core-js 实现按需引入。
安装 core-js，记得添记到 dependencies 中去，因为打包后需要引入

npm install --save core-js
1

@babel/preset-env预设中自带了按需引入 polyfill 的功能，只需要在配置中使用 "useBuiltIns": "usage" 参数即可实现按需引入对应 core-js下的 polyfill
更改 babel 配置

{
  "presets": [
    [
      "@babel/preset-env",
      {
        "useBuiltIns": "usage",
        "corejs": 3
      }
    ]
  ]
}

# core-js@2 和 core-js@3 的区别

有些项目配置中需要指定 core-js 的版本是 2 还是 3 ，这两者的区别是什么呢？
core-js@2版本已经停止更新了，新的 **API**** **兼容会更新至 core-js@3, 之前core-js@2被@babel/polyfill中引用做兼容, 而core-js@3做了一些优化，如体积、引用关系等，推荐使用 core-js@3

# transform-runtime 搭配 babel/runtime

前面讲到@babel/preset-env配合core-js使用，会有重复定义辅助函数的问题，而且也没有解决全局方法修改的问题，还有一种方式就是使用@babel/plugin-transform-runtime搭配@babel/runtime-corejs3。

# @babel/plugin-transform-runtime

@babel/plugin-transform-runtime是一个按需引入 polyfill的插件，它会找出需要使用polyfill和辅助函数的地方，按需引入@babel/runtime-corejs3下面的 polyfill和辅助函数，也不会修改全局方法
安装，需要安装到 dependencies 中

npm install --save-dev @babel/plugin-transform-runtime
1

# @babel/runtime-corejs3

@babel/runtime-corejs3是一个 polyfill代码库，它引用了core-js@3，并且可以配合 @babel/plugin-transform-runtime来完成按需引入polyfill和辅助函数。
安装，需要安装到 dependencies 中

npm install --save @babel/runtime-corejs3
1

修改配置

{
  "presets": [["@babel/preset-env"]],
  "plugins": [
    [
      "@babel/plugin-transform-runtime",
      {
        "corejs": 3
      }
    ]
  ]
}

# 总结

语法转换可以使用常用的 presets来实现转译。
API可以使用以下几种polyfill方式来转译

  1. @babel/polyfill
    没有按需引入、体积大、修改全局方法、重复定义辅助函数，不推荐
  2. core-js 搭配 @babel/preset-env
    有按需引入、会重复定义辅助函数、修改全局方法，不推荐
  3. @babel/runtime-corejs3搭配 @babel/plugin-transform-runtime
    有按需引入，不会重复定义辅助函数，不修改全局方法，推荐
