# 什么是事件循环？

事件循环是指 JavaScript 为了解决单线程的低效率问题而创建出来的任务执行机制。

# JavaScript 为什么是单线程？

单线程的意思是同一时间只能做一件事情，JavaScript 执行代码的线程只有一个，如果有非常耗时的任务，后面的任务需要等待前一个任务执行完毕后才会执行，这就是单线程低效的原因。 那么为什么 JavaScript 要设计成单线程的一门语言呢？ 因为 JavaScript 是作为浏览器的脚本语言而设计的，主要目的是为了和用户进行交互，例如操作 Dom ，这些操作如果使用多线程的话会带来很复杂的同步问题，如果多线程同时操作 Dom 的话该如何处理？所以 JavaScript 是单线程的。 但是单线程的劣势也很明显，执行任务效率低下，任务需要一个接一个完成，所以事件循环机制就是为了解决这些问题而存在的。

# 执行机制

为了解决单线程的问题 JavaScript 将任务分为同步Synchronous与异步Asynchronous 两种。

# 同步

同步任务也就是阻塞的，需要等待上一个任务完成才能开始下一个任务。

let num = 1
console.log(num) // 1
num = 2
console.log(num) // 2

# 异步

异步任务也就是非阻塞的，执行异步任务时会注册一个该任务的回调函数，任务执行完成后如果主线程中还有同步任务会等待同步任务全部完成后再执行该任务的回调函数。

let num = 1
setTimeout(() => {
  num = 2
  console.log(2) // 2
}, 50)
for (let i = 0; i < 1000000000; i++) {}
console.log(num) // 1
